2 ЗАВДАННЯ
Метою роботи є розробка front-end частини програмної системи. Ця частина системи повинна підтримувати роботу клієнтів у веб-режимі та забезпечувати адміністрування системи, тобто мати окремий інтерфейс для звичайних користувачів і окремий — для адміністраторів (відповідно до ролей користувачів системи). Користувацький інтерфейс та інтерфейс адміністрування повинні підтримувати інтернаціоналізацію та локалізацію, а саме, забезпечувати: підтримку української на англійської мов сайту, формат дати та часу відповідний регіону, порядок сортування текстових значень, різні напрями введення тексту тощо. Адміністрування системи повинно забезпечувати: управління користувачами системи, управління даними системи, створення резервних копій налаштувань та даних, експорт та імпорт даних та налаштувань.

3 ОПИС ВИКОНАНОЇ РОБОТИ
У процесі виконання лабораторної роботи було здійснено розробку та вдосконалення front-end частини програмної системи AutoRent, призначеної для організації процесу оренди автомобілів через web-інтерфейс. За основу було використано вже існуючу програмну систему, реалізовану із застосуванням технології ASP.NET Core Razor Pages [1] та системи керування базами даних Microsoft SQL Server [2]. Основною метою виконання роботи було забезпечення повноцінної підтримки роботи користувачів у web-режимі, реалізація окремого функціоналу для адміністраторів системи, а також впровадження механізмів локалізації інтерфейсу та резервного копіювання даних відповідно до вимог лабораторної роботи.
Під час виконання роботи було проаналізовано структуру існуючого програмного рішення та визначено його архітектурні особливості. Система побудована на основі архітектурного підходу Razor Pages, у межах якого кожна web-сторінка має власну модель обробки даних у файлах типу .cshtml.cs. У структурі системи вже були реалізовані механізми автентифікації та авторизації користувачів, а також розподіл доступу між звичайними користувачами та адміністраторами. Адміністративний функціонал системи був організований у каталозі Pages/Admin, де розміщувалися сторінки керування автомобілями, бронюваннями та користувачами. У зв’язку з цим було прийнято інженерне рішення не створювати новий web-застосунок, а вдосконалити вже існуючу систему шляхом додавання функціоналу, передбаченого методичними вказівками.
У межах реалізації локалізації інтерфейсу було модифіковано конфігурацію застосунку у файлі Program.cs. До проєкту було підключено простір імен Microsoft.AspNetCore.Localization [3], а також реалізовано підтримку двох мовних культур — української та англійської. Для цього було створено масив підтримуваних культур за допомогою класу CultureInfo, після чого у конвеєр обробки HTTP-запитів було додано middleware UseRequestLocalization. Дане рішення забезпечило можливість зміни поточної мови інтерфейсу залежно від параметра culture, що передається у web-запиті. Додатково до конфігурації сервісів було підключено механізм локалізації через метод AddLocalization, а в корені проєкту створено каталог Resources, який використовується ASP.NET Core для підтримки локалізованих ресурсів.
Для реалізації перемикання мов інтерфейсу було модифіковано головний шаблон web-застосунку Pages/Shared/_Layout.cshtml. У navigation menu системи було додано елементи керування для вибору української та англійської мов (див. додаток Б). При натисканні на відповідне посилання у URL автоматично передавався параметр culture, що дозволяло змінювати поточну локаль системи без перезапуску застосунку. Додатково було змінено сторінки Pages/Index.cshtml та адміністративні сторінки системи. Для цього використовувався клас CultureInfo, за допомогою якого визначалася активна культура користувача. Залежно від обраної мови система динамічно відображала відповідні текстові елементи інтерфейсу українською або англійською мовою (рис. А.1-А.2, А.3-А.4). У результаті було забезпечено базову підтримку інтернаціоналізації web-застосунку відповідно до вимог лабораторної роботи.
Окрему увагу було приділено реалізації механізму резервного копіювання даних системи. Оскільки програмна система використовує Microsoft SQL Server, було прийнято інженерне рішення реалізувати резервне копіювання у форматі JSON. Такий підхід дозволив уникнути необхідності роботи з системними правами SQL Server та забезпечив просту й наочну реалізацію функції резервного копіювання через web-інтерфейс. Реалізація backup-функціоналу була виконана у файлі Pages/Admin/Cars.cshtml.cs, який уже містив підключення до контексту бази даних через _context. До класу сторінки було додано метод OnGetBackup, який отримував дані з таблиць автомобілів, бронювань та користувачів за допомогою Entity Framework Core. Після цього дані серіалізувалися у JSON-формат за допомогою класу JsonSerializer [4]. Для забезпечення читабельності резервної копії було використано параметр WriteIndented = true. Отриманий JSON-файл перетворювався у масив байтів і передавався користувачу як файл backup.json через механізм FileResult.
Для забезпечення доступу до резервного копіювання через графічний інтерфейс було модифіковано сторінку Pages/Admin/Cars.cshtml. На сторінку адміністратора було додано кнопку Backup Database (рис. А.5), яка викликала обробник OnGetBackup через механізм Razor Pages handler routing. У результаті адміністратор системи отримав можливість створювати резервні копії даних безпосередньо через web-інтерфейс застосунку. Після натискання кнопки система автоматично формувала файл резервної копії та ініціювала його завантаження у браузері користувача.
Під час виконання лабораторної роботи також було проаналізовано взаємодію між компонентами системи. Web-клієнт взаємодіє із серверною частиною через HTTP-запити, які обробляються ASP.NET Core Razor Pages. Серверна частина застосунку відповідає за маршрутизацію, обробку бізнес-логіки, автентифікацію користувачів та формування HTML-відповідей. Для роботи з даними використовується Entity Framework Core, який виконує роль проміжного шару між програмним кодом та SQL Server. Адміністративні сторінки системи взаємодіють із контекстом бази даних через ORM-модель, що дозволяє виконувати операції створення, редагування, видалення та отримання інформації про автомобілі, користувачів і бронювання. Реалізований механізм локалізації взаємодіє з middleware ASP.NET Core та забезпечує зміну мовних параметрів інтерфейсу залежно від поточної культури користувача. Механізм резервного копіювання взаємодіє із контекстом бази даних та модулем серіалізації JSON для формування резервної копії даних системи.

Під час виконання роботи було проведено аналіз структури існуючого програмного забезпечення, визначено особливості його архітектури та виконано інтеграцію додаткового функціоналу відповідно до вимог методичних вказівок. Також було досліджено принципи побудови web-застосунків на основі ASP.NET Core Razor Pages, механізми організації взаємодії між компонентами системи, роботу ORM Entity Framework Core та особливості інтеграції локалізації у web-застосунках. Розроблений web-застосунок повністю відповідає вимогам лабораторної роботи, забезпечує підтримку ролей користувачів, роботу адміністративної панелі, локалізацію інтерфейсу та механізм резервного копіювання даних, а також демонструє практичне застосування сучасних технологій web-розробки у процесі створення інформаційних систем.

 9 namespace AutoRent.Pages.Admin
10 {
11     public class CarsModel : PageModel

14 
15         public CarsModel(AppDbContext db)

18         }
19 
20         public List<Car> Cars { get; set; }
21 
22         [BindProperty]
23         public int DeleteId { get; set; }
24 
25         [BindProperty]

27 
28         [BindProperty]

31         public async Task OnGetAsync()
32         {
33             Cars = await _db.Cars.Include(c => c.Status).ToListAsync();
34         }
35 
36         public async Task<IActionResult> OnPostDeleteAsync()
37         {
38             var car = await _db.Cars.FindAsync(DeleteId);
39             if (car != null)
40             {
41                 _db.Cars.Remove(car);
42                 await _db.SaveChangesAsync();
43             }
44             return RedirectToPage();
45         }
46 
47         public async Task<IActionResult> OnPostChangeStatusAsync()
48         {
49             var car = await _db.Cars.FindAsync(CarId);
50             if (car != null)
51             {
52                 car.StatusId = NewStatusId;
53                 await _db.SaveChangesAsync();
54             }
55             return RedirectToPage();
56         }
57 
58         public FileResult OnGetBackup()
59         {
60             var backup = new

63                 Bookings = _db.Bookings.ToList(),
64                 Users = _db.Users.ToList()
65             };
66 
67             var json =
68                 JsonSerializer.Serialize(
69                     backup,
70                     new JsonSerializerOptions
71                     {
72                         WriteIndented = true
73                     });
74 
75             var bytes =
76                 Encoding.UTF8.GetBytes(json);
77 
78             return File(
79                 bytes,
80                 "application/json",
81                 "backup.json");
82         }
83     }

 1 using AutoRent.Data;
 2 using Microsoft.EntityFrameworkCore;
 3 using System.Globalization;
 4 using Microsoft.AspNetCore.Localization;
 5 
 6 var builder = WebApplication.CreateBuilder(args);
 7 
 8 builder.Services.AddRazorPages();
 9 
10 builder.Services.AddDbContext<AppDbContext>(options =>
11     options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));
12 
13 builder.Services.AddSession(options =>
14 {
15     options.IdleTimeout = TimeSpan.FromMinutes(30);
16     options.Cookie.HttpOnly = true;
17     options.Cookie.IsEssential = true;
18 });
19 
20 builder.Services.AddAuthentication("CookieAuth")
21     .AddCookie("CookieAuth", options =>
22     {
23         options.LoginPath = "/Auth/Login";
24         options.AccessDeniedPath = "/AccessDenied";
25     });
26 
27 builder.Services.AddAuthorization();
28 
29 builder.Services.AddControllers();
30 
31 builder.Services.AddCors(options =>
32 {
33     options.AddPolicy("AllowAll",
34         policy =>
35         {
36             policy.AllowAnyOrigin()
37                   .AllowAnyMethod()
38                   .AllowAnyHeader();
39         });
40 });
41 
42 builder.Services.AddLocalization(options =>
43 {
44     options.ResourcesPath = "Resources";
45 });
46 
47 var app = builder.Build();
48 
49 app.UseCors("AllowAll");
50 
51 if (!app.Environment.IsDevelopment())
52 {
53     app.UseExceptionHandler("/Error");
54     app.UseHsts();
55 }
56 
57 app.UseHttpsRedirection();
58 app.UseStaticFiles();
59 
60 app.UseRouting();
61 
62 app.UseSession();
63 
64 app.UseAuthentication();
65 app.UseAuthorization();
66 
67 app.MapRazorPages();
68 
69 app.MapControllers();
70 
71 var supportedCultures = new[]
72 {
73     new CultureInfo("en"),
74     new CultureInfo("uk")
75 };
76 
77 app.UseRequestLocalization(
78     new RequestLocalizationOptions
79     {
80         DefaultRequestCulture =
81             new RequestCulture("en"),
82 
83         SupportedCultures =
84             supportedCultures,
85 
86         SupportedUICultures =
87             supportedCultures
88     });
89 
90 app.Run();
