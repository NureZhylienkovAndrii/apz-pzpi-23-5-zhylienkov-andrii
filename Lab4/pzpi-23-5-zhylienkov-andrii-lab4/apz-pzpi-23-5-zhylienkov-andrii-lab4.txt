3 ОПИС ВИКОНАНОЇ РОБОТИ
У процесі виконання лабораторної роботи було здійснено масштабування backend-частини програмної системи AutoRent з метою забезпечення стабільної роботи застосунку при збільшенні навантаження та одночасному обслуговуванні великої кількості користувачів. За основу було використано існуючу клієнт-серверну систему для оренди автомобілів, реалізовану на платформі ASP.NET Core 8 із використанням Razor Pages, Entity Framework Core та Microsoft SQL Server. На початковому етапі було проведено детальний аналіз структури програмного забезпечення та визначено його архітектурні особливості. У ході аналізу було встановлено, що система побудована за монолітним принципом та містить серверну частину, представлену web-застосунком ASP.NET Core, моделі даних у каталозі Models, контекст бази даних у каталозі Data, Razor Pages для реалізації користувацького інтерфейсу у каталозі Pages, а також статичні ресурси у каталозі wwwroot. Крім того, система містила реалізований механізм автентифікації та авторизації користувачів, адміністративну панель, функціонал бронювання автомобілів, локалізацію інтерфейсу та механізм резервного копіювання даних, які були створені під час попередніх лабораторних робіт.
Для реалізації масштабування backend-частини було обрано горизонтальний підхід, відповідно до якого декілька копій серверного застосунку одночасно виконують однакові функції та обробляють запити користувачів паралельно. Такий підхід було обрано через його простоту реалізації, можливість демонстрації принципу балансування навантаження та відповідність вимогам лабораторної роботи. Для контейнеризації серверної частини системи було використано технологію Docker. У корені проєкту було створено файл Dockerfile, у якому було реалізовано процес автоматичного створення контейнера із серверним застосунком ASP.NET Core. На першому етапі Docker-збірки використовувався образ mcr.microsoft.com/dotnet/sdk:8.0 для компіляції та публікації проєкту, після чого результати збірки копіювалися у фінальний runtime-образ mcr.microsoft.com/dotnet/aspnet:8.0. У Dockerfile було визначено робочу директорію контейнера, відкрито порт 8080, а також налаштовано запуск застосунку через файл AutoRent.dll. Дане рішення дозволило створити ізольоване середовище виконання серверної частини та забезпечити можливість одночасного запуску декількох копій web-застосунку.
Для організації взаємодії між декількома екземплярами серверного застосунку було використано систему Docker Compose [1]. У корені проєкту було створено файл docker-compose.yml, у якому описано конфігурацію запуску контейнерів. У межах конфігурації було створено окремі сервіси app1, app2 та app3, кожен із яких представляв окрему копію ASP.NET Core серверу. Усі сервери створювалися на основі одного Dockerfile та працювали паралельно. Додатково у docker-compose конфігурації було створено окремий контейнер із веб-сервером Nginx [2], який виконував функцію балансувальника навантаження. Для цього було використано офіційний Docker-образ nginx:latest, а також налаштовано автоматичне підключення власного конфігураційного файлу nginx.conf.
Для реалізації балансування навантаження було створено каталог nginx, у якому розміщувався файл nginx.conf. У межах конфігурації Nginx було створено блок upstream autorent_servers, який містив перелік серверів app1:8080, app2:8080 та app3:8080. Далі було налаштовано web-сервер Nginx на прослуховування порту 80 та переспрямування всіх HTTP-запитів до пулу серверів через директиву proxy_pass. У результаті запити користувачів автоматично розподілялися між декількома копіями backend-застосунку, що забезпечило реалізацію горизонтального масштабування системи.
Під час інтеграції контейнеризації було виявлено проблему підключення серверного застосунку до локальної бази даних Microsoft SQL Server усередині Docker-контейнерів. Оскільки контейнеризоване середовище не мало доступу до локального SQL Server через адресу localhost, було прийнято інженерне рішення не використовувати базу даних у процесі навантажувального тестування. Для цього у файлі Program.cs було реалізовано окремий тестовий endpoint /health, який не використовував Entity Framework Core, не виконував звернення до бази даних та повертав просту текстову відповідь "Server is working" (додаток Б). Додатково у Program.cs було виконано налаштування прослуховування порту 8080 через метод app.Urls.Add("http://0.0.0.0:8080"), що забезпечило коректну роботу застосунку всередині Docker-контейнерів та можливість взаємодії з Nginx Load Balancer.
Після реалізації контейнеризації та балансування навантаження було проведено навантажувальне тестування системи за допомогою програмного інструмента Apache JMeter [3]. Для цього було створено тестовий сценарій, який виконував HTTP GET-запити до endpoint /health через адресу http://localhost:8080/health. У конфігурації тестування було налаштовано 100 одночасних користувачів із поступовим запуском протягом 10 секунд та багаторазовим повторенням запитів. Для аналізу результатів використовувалися компоненти Summary Report, Graph Results та View Results Tree, що дозволило оцінити середній час відповіді сервера, максимальний час відповіді, кількість запитів за секунду та стабільність роботи системи.
У процесі тестування було проведено порівняння роботи системи у двох режимах (додаток А). Спочатку застосунок запускався лише з одним сервером app1, після чого виконувалося навантажувальне тестування. Далі конфігурацію було змінено таким чином, щоб одночасно працювали три сервери app1, app2 та app3, після чого тестування повторювалося. У результаті було встановлено, що при використанні трьох серверів максимальний час відповіді зменшився з 50 мс до 8 мс, а стандартне відхилення часу відповіді зменшилося з 2.36 до 1.20. Отримані результати підтвердили підвищення стабільності роботи backend-системи при використанні горизонтального масштабування та балансування навантаження. Хоча показник throughput залишився майже незмінним, це пояснюється тим, що endpoint /health не виконував складних обчислень або взаємодії з базою даних, а отже навіть один сервер міг ефективно обробляти навантаження.

У результаті виконання лабораторної роботи було реалізовано повноцінну систему горизонтального масштабування backend-частини програмної системи AutoRent із використанням Docker-контейнеризації, балансування навантаження через Nginx та навантажувального тестування за допомогою Apache JMeter.
