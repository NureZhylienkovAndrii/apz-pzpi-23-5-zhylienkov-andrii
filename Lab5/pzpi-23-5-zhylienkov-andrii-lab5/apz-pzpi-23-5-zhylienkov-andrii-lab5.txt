3 ОПИС ВИКОНАНОЇ РОБОТИ
У межах виконання лабораторної роботи було підготовлено завершену презентацію програмної системи AutoRent, яка розроблялася протягом двох семестрів у процесі виконання попередніх лабораторних робіт. Основною метою даного етапу було узагальнення результатів проєктування та реалізації програмної системи, демонстрація функціональних можливостей створеного програмного забезпечення, а також підготовка матеріалів для представлення готового програмного продукту у форматі демонстрації та технічної презентації. На даному етапі було виконано структуризацію всіх компонентів системи, узгодження функціональних модулів між собою, підготовку демонстраційних матеріалів та оформлення фінального звіту.
У процесі виконання роботи було проаналізовано повну структуру програмної системи AutoRent, яка реалізована за клієнт-серверною архітектурою. Серверна частина системи була розроблена на платформі ASP.NET Core 8 із використанням Razor Pages та Entity Framework Core. Backend-частина системи забезпечує обробку HTTP-запитів, взаємодію із базою даних, авторизацію та автентифікацію користувачів, виконання бізнес-логіки системи оренди автомобілів, а також підтримку REST API для мобільного застосунку. У ролі системи керування базами даних використовується Microsoft SQL Server, взаємодія з яким реалізована через ORM Entity Framework Core. У системі було реалізовано моделі користувачів, автомобілів, бронювань та ролей, а також логіку їх взаємодії через контекст бази даних ApplicationDbContext.
Під час підготовки презентації було детально проаналізовано функціональні можливості web-застосунку. У системі реалізовано механізм реєстрації нових користувачів із перевіркою коректності введених даних, механізм авторизації через систему автентифікації ASP.NET Core Identity, а також систему розподілу ролей між звичайними користувачами та адміністраторами. Було реалізовано окремий адміністративний інтерфейс, який забезпечує управління користувачами, автомобілями та бронюваннями. Адміністративна панель дозволяє змінювати стан автомобілів, контролювати активні бронювання та виконувати базові операції адміністрування системи.
У ході роботи було також продемонстровано функціонал перегляду автомобілів та їх бронювання. Користувачі системи мають можливість переглядати список доступних транспортних засобів із зазначенням бренду, моделі, року випуску та вартості оренди за день. Для реалізації даного функціоналу використовувалися Razor Pages та серверна генерація HTML-сторінок. Інформація про автомобілі отримується із бази даних через Entity Framework Core та передається до представлень через PageModel-класи. Також у системі реалізовано механізм створення бронювань із подальшим збереженням інформації у базі даних.
Окрему увагу під час виконання роботи було приділено мобільному застосунку, який було розроблено для платформи Android із використанням мови програмування Kotlin. Мобільний клієнт взаємодіє із серверною частиною системи через REST API, реалізований у backend-застосунку ASP.NET Core. Для виконання HTTP-запитів та обробки JSON-відповідей використовувалася бібліотека Retrofit. У мобільному застосунку реалізовано отримання списку автомобілів із серверної частини системи та їх відображення у вигляді списку на екрані Android-застосунку. Для демонстрації роботи мобільного клієнта використовувався Android Emulator у середовищі Android Studio.
У процесі виконання лабораторної роботи було також продемонстровано реалізовану підтримку локалізації web-застосунку. У системі було налаштовано підтримку української та англійської мов інтерфейсу із можливістю перемикання мови користувачем. Для реалізації локалізації використовувалися механізми ASP.NET Core Localization та ресурсні файли із текстовими значеннями інтерфейсу. Було реалізовано коректне відображення локалізованих елементів навігації, кнопок та текстових повідомлень.
Додатково у межах роботи було продемонстровано реалізацію резервного копіювання даних системи. Було створено функціонал формування резервних копій даних у форматі JSON із можливістю експорту інформації про користувачів, автомобілі та бронювання. Для цього було створено окрему модель BackupModel та реалізовано логіку серіалізації даних із бази даних у JSON-файл. Даний функціонал інтегровано до адміністративної частини системи.
На завершальному етапі роботи було виконано демонстрацію масштабування backend-частини системи. Для цього серверний застосунок було контейнеризовано за допомогою Docker. Було створено Dockerfile для автоматизованої збірки та запуску ASP.NET Core-застосунку у контейнері. Для запуску декількох екземплярів backend-сервера використовувався Docker Compose, а балансування навантаження між серверами виконувалося через Nginx Load Balancer. У конфігурації Nginx було налаштовано upstream-групу серверів та проксіювання HTTP-запитів між декількома контейнерами застосунку.
Для перевірки ефективності масштабування було проведено навантажувальне тестування системи за допомогою Apache JMeter. Під час тестування виконувалося порівняння роботи системи при використанні одного серверного екземпляра та трьох серверних екземплярів. Було проаналізовано показники максимального часу відповіді, стандартного відхилення та пропускної здатності системи. Результати тестування продемонстрували зменшення максимального часу відповіді системи при використанні горизонтального масштабування backend-частини.
У процесі підготовки фінальної презентації було створено структуровані слайди (додаток Б), які демонструють архітектуру системи, основні функціональні можливості, взаємодію між компонентами, роботу мобільного застосунку, механізми локалізації та резервного копіювання, а також реалізацію масштабування backend-інфраструктури. Окремо було підготовлено демонстраційний відеоролик (додаток А) із покроковою демонстрацією роботи основних компонентів системи, включаючи web-застосунок, мобільний клієнт, адміністративну панель та результати навантажувального тестування. У результаті виконання лабораторної роботи було підготовлено завершену презентацію програмної системи AutoRent, яка демонструє реалізацію повноцінної багатокомпонентної системи оренди автомобілів із підтримкою web та mobile доступу, адміністрування, локалізації, резервного копіювання даних та масштабування backend-інфраструктури.
